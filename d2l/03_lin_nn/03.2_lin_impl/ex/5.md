# 在 `squared_loss` 中使用 `reshape` 的目的

在 `squared_loss` 函数中使用 `reshape`（或者 `reshape_as`）的目的是：

**保证预测值和真实标签的形状一致。**

否则在计算损失时，可能发生：

- 维度不匹配
- 广播（broadcast）错误
- 计算结果不符合预期

---

## 1. 《动手学深度学习》中的代码

通常代码类似：

```python
def squared_loss(y_hat, y):
    return (y_hat - y.reshape(y_hat.shape)) ** 2 / 2
```

这里关键是：

```python
y.reshape(y_hat.shape)
```

---

## 2. 为什么需要 reshape？

假设：

预测值：

```python
y_hat.shape = (4, 1)
```

真实值：

```python
y.shape = (4,)
```

虽然它们都表示 4 个样本，但：

- 一个是二维张量
- 一个是一维张量

形状不同。

---

## 3. 不 reshape 会发生什么？

如果直接：

```python
y_hat - y
```

根据广播机制，PyTorch 会自动扩展维度。结果可能变成：

```python
(4, 1) - (4,)
→ (4, 4)
```

这就错了。

本来我们希望：

```python
[y1_hat - y1,
 y2_hat - y2,
 ...]
```

结果却变成：

```python
[[y1_hat - y1, y1_hat - y2, ...],
 [y2_hat - y1, y2_hat - y2, ...],
 ...]
```

损失完全错误。

---

## 4. reshape 的作用

```python
y.reshape(y_hat.shape)
```

会把 `(4,)` 变成 `(4, 1)`。

于是：

- 两者形状一致
- 元素一一对应
- 正确计算平方损失

---

## 5. 平方损失公式

平方损失：

$$
\ell(y, \hat{y}) = \frac{1}{2}(y - \hat{y})^2
$$

这里要求 $y$ 和 $\hat{y}$ 必须逐元素对应。reshape 正是为了保证这一点。

---

## 6. 举个具体例子

### 不 reshape

```python
import torch

y_hat = torch.tensor([[1.0], [2.0], [3.0]])
y = torch.tensor([1.0, 2.0, 3.0])

print((y_hat - y).shape)
```

输出：

```
torch.Size([3, 3])
```

错误。

### reshape 后

```python
y = y.reshape(y_hat.shape)

print((y_hat - y).shape)
```

输出：

```
torch.Size([3, 1])
```

正确。

---

## 7. 总结

在 `squared_loss` 函数中使用 `reshape`，是为了保证真实标签 `y` 与预测值 `y_hat` 具有相同的形状。这样可以避免由于广播机制导致的维度错误，使平方损失能够按照元素一一对应的方式正确计算。
