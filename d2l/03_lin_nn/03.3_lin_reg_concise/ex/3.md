# 3.如何访问线性回归的梯度？


在深度学习框架中，"访问梯度"通常指在反向传播后查看模型参数（权重和偏置）相对于损失函数的偏导数。对于线性回归，这些参数是权重 \( w \) 和偏置 \( b \)。下面以 PyTorch 为主，TensorFlow 为辅，说明具体操作。

---

## 1. 在 PyTorch 中访问线性回归的梯度

### 核心步骤
1. 定义参数为可训练张量（`requires_grad=True` 或 `nn.Parameter`）。
2. 执行前向传播并计算损失。
3. 调用 `loss.backward()` 计算梯度。
4. 通过参数的 `.grad` 属性访问梯度。

### 示例代码（使用 `nn.Linear`）
```python
import torch
import torch.nn as nn

# 构建一个单层线性回归模型
model = nn.Linear(in_features=1, out_features=1)  # w 和 b 已包含 requires_grad=True

# 模拟数据
x = torch.tensor([[1.0], [2.0], [3.0]])
y_true = torch.tensor([[2.0], [4.0], [6.0]])

# 定义损失（例如 MSE）
criterion = nn.MSELoss()

# 前向传播
y_pred = model(x)
loss = criterion(y_pred, y_true)

# 反向传播
loss.backward()

# 访问梯度
print("权重 w 的梯度:", model.weight.grad)
print("偏置 b 的梯度:", model.bias.grad)
```

### 输出示例（梯度值取决于初始化和数据）
```
权重 w 的梯度: tensor([[-4.1234]])
偏置 b 的梯度: tensor([-2.5678])
```

### 手动管理参数时（不使用 `nn.Linear`）
```python
w = torch.randn(1, requires_grad=True)
b = torch.randn(1, requires_grad=True)

y_pred = x @ w + b
loss = criterion(y_pred, y_true)
loss.backward()

print(w.grad, b.grad)
```

### 注意事项
- 梯度会**累积**，多次反向传播前需调用 `optimizer.zero_grad()` 或手动置零（`w.grad = None`）。
- 只有 `requires_grad=True` 的叶节点张量才会有 `.grad` 属性。
- 如果使用 `torch.no_grad()` 上下文或 `.detach()`，则不会记录梯度。

---

## 2. 在 TensorFlow/Keras 中访问梯度

使用 `tf.GradientTape` 记录前向操作，然后显式计算梯度。

### 示例代码
```python
import tensorflow as tf

# 定义变量
w = tf.Variable(tf.random.normal([1, 1]))
b = tf.Variable(tf.random.normal([1]))

x = tf.constant([[1.0], [2.0], [3.0]])
y_true = tf.constant([[2.0], [4.0], [6.0]])

with tf.GradientTape() as tape:
    y_pred = x @ w + b
    loss = tf.reduce_mean(tf.square(y_pred - y_true))  # MSE

# 计算梯度
dw, db = tape.gradient(loss, [w, b])
print("w 梯度:", dw.numpy())
print("b 梯度:", db.numpy())
```

在 Keras 自定义训练循环中也可以用同样方式访问。

---

## 3. 线性回归梯度的数学形式（MSE 损失）

若损失为 $ ( L = \frac{1}{N} \sum_{i=1}^N (y_i - (w x_i + b))^2 ) $，则梯度为：

$$
\frac{\partial L}{\partial w} = -\frac{2}{N} \sum_{i=1}^N x_i (y_i - \hat{y}_i)
$$

$$
\frac{\partial L}{\partial b} = -\frac{2}{N} \sum_{i=1}^N (y_i - \hat{y}_i)
$$

框架自动完成这些计算，我们通过 `.grad` 或 `tape.gradient()` 得到的就是这些数值。

## 4. 与之前问题的关联

如果已将损失替换为 Huber 损失（公式 3.3.1），访问梯度的代码完全不变——只需将 `criterion` 换成你定义的 Huber 损失，反向传播后 `model.weight.grad` 就会给出 Huber 损失对权重的梯度，模型能够对抗异常值的影响。
