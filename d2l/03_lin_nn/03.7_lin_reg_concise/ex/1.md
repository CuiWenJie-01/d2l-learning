# 1.尝试调整超参数，例如批量大小、迭代周期数和学习率，并查看结果。
在 softmax 回归模型中，超参数（如批量大小、迭代周期数、学习率）的选择会显著影响训练过程和最终性能。下面以常见的分类数据集（如 Fashion-MNIST）为例，解释如何调整这些超参数以及应观察哪些结果。

---

### 1. 学习率（learning rate）

学习率控制每次参数更新的步长。

- **过大**：损失值振荡甚至发散，准确率无法提升或出现 NaN。
- **过小**：收敛极慢，需要很多周期才能达到较好的准确率。
- **合适**：损失平滑下降，准确率稳定上升。

**实验建议**：
- 常用范围：0.1, 0.01, 0.001, 0.0001。
- 观察训练损失曲线：若下降平缓则尝试增大；若剧烈震荡则减小。
- 最佳学习率通常使损失在几十个迭代步内明显下降。

---

### 2. 批量大小（batch size）

批量大小决定每次参数更新使用的样本数。

- **小批量（如 16, 32, 64）**：
  - 梯度估计噪声大，损失曲线波动明显，但可能帮助跳出局部极小。
  - 训练速度较慢（单位时间内迭代步数多，但每步计算量小）。
- **大批量（如 256, 512, 1024）**：
  - 梯度更稳定，损失曲线平滑。
  - 可以充分利用矩阵运算加速，但单步计算量大。
  - 可能过拟合（泛化能力稍差），且容易陷入尖锐极小值。

**实验建议**：
- 批量大小通常设为 2 的幂（符合内存对齐）。
- 在相同迭代周期下，大批量比小批量每个 epoch 的迭代步数更少，因此总计算次数少。
- 观察训练和测试准确率的差距：若大批量导致测试准确率下降，可尝试降低批量或添加正则化。

---

### 3. 迭代周期数（epochs）

周期数决定模型看到完整数据集的次数。

- **欠拟合**：周期太少，训练和测试准确率都较低，损失未收敛。
- **过拟合**：周期太多，训练准确率继续上升，但测试准确率停滞或下降（软max回归过拟合通常不严重，但若模型复杂或数据少也可能发生）。
- **合适**：当测试准确率不再提升甚至下降时，应提前停止。

**实验建议**：
- 从 10~20 个 epoch 开始，观察损失曲线是否平坦。
- 可以绘制训练/测试准确率随 epoch 变化的曲线，选择验证集性能最高的点。

---

### 典型实验设计与结果解读

假设使用 Fashion-MNIST，softmax 回归，学习率 0.01，批量 256，epoch 10，测试准确率约 0.84。

#### 调整示例：

| 超参数调整 | 现象 | 解释 |
| --- | --- | --- |
| 学习率 → 0.1 | 训练损失骤降后迅速发散（NaN） | 步长太大，参数跳过最优区域 |
| 学习率 → 0.001 | 训练损失下降极慢，10 epoch 后准确率仅 0.75 | 收敛太慢，需要更多 epoch 或增大学习率 |
| 批量大小 → 64 | 损失曲线更波动，但最终准确率相近 | 梯度噪声大，但平均方向相同 |
| 批量大小 → 2048 | 训练损失平滑，但测试准确率稍降（0.83） | 泛化略差，可尝试增加正则化 |
| epoch → 5 | 训练损失未完全收敛，准确率 0.80 | 欠拟合 |
| epoch → 30 | 训练准确率 0.85，测试仍为 0.84 | 已收敛，更多 epoch 无改善 |

### 代码示例（PyTorch）

```python
import torch
from torch import nn
from torch.utils.data import DataLoader
from torchvision import datasets, transforms

# 超参数待调整
batch_size = 256
lr = 0.01
epochs = 10

# 加载数据
transform = transforms.ToTensor()
train_dataset = datasets.FashionMNIST(root='./data', train=True, download=True, transform=transform)
test_dataset = datasets.FashionMNIST(root='./data', train=False, download=True, transform=transform)
train_loader = DataLoader(train_dataset, batch_size=batch_size, shuffle=True)
test_loader = DataLoader(test_dataset, batch_size=batch_size, shuffle=False)

# 模型
model = nn.Sequential(nn.Flatten(), nn.Linear(28*28, 10))
loss_fn = nn.CrossEntropyLoss()
optimizer = torch.optim.SGD(model.parameters(), lr=lr)

# 训练并记录
for epoch in range(epochs):
    model.train()
    for X, y in train_loader:
        pred = model(X)
        loss = loss_fn(pred, y)
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
    # 测试
    model.eval()
    correct = 0
    with torch.no_grad():
        for X, y in test_loader:
            pred = model(X)
            correct += (pred.argmax(1) == y).sum().item()
    acc = correct / len(test_dataset)
    print(f"Epoch {epoch+1}, test acc: {acc:.4f}")
```

运行后，你可以修改 `batch_size`、`lr`、`epochs` 观察输出变化。

---

### 结论

- **学习率**最重要，需优先调节。
- **批量大小**影响训练速度和梯度稳定性，可根据内存选择。
- **迭代周期数**决定是否收敛，通常通过验证集早停。

通过系统调整这些超参数，你可以找到在给定模型和数据集上表现最佳的组合。
